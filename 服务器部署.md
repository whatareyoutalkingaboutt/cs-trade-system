# CS 饰品全量监控系统 - 服务器部署与极限性能调优指南

本手册涵盖了从本地代码推送到服务器部署的完整流程。针对目标服务器环境（**CPU 2核, 内存 2GB, NVMe 40GB, 10M上行/100M下行带宽**），提供了一套极限压榨硬件的防爆栈部署方案。

---

## 第一部分：部署流程 (Local -> GitHub -> Server)

### 1. 本地代码准备
在推送前，确保敏感信息已排除。
1. **检查 `.gitignore`**: 确保包含 `venv/`, `__pycache__/`, `.env`, `*.log`, `data/`。
2. **生成依赖清单**: `pip freeze > requirements.txt`
3. **推送到远程**: 提交代码并 Push 至 GitHub 私有仓库。

### 2. 服务器环境初始化
服务器选用 **Ubuntu 22.04 LTS**。
```bash
# 1. 更新并安装环境
sudo apt update && sudo apt install -y git docker.io docker-compose-plugin

# 2. 克隆项目并配置
git clone https://github.com/你的用户名/cs-item-scraper.git
cd cs-item-scraper
cp .env.example .env  # 填入数据库密码及必要配置
```

### 3. Docker 一键启动
```bash
sudo docker compose up -d --build
sudo docker compose logs -f api ws-server frontend celery-worker celery-beat
```
